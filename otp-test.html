Troubleshooting

  1. Make sure the file name ends with .html (not .txt or .rtf).
  2. On Windows Notepad: use Save as → All Files and name otp-test.html and select UTF-8 encoding.
  3. On macOS TextEdit: convert to Plain Text first (Format → Make Plain Text), then save with .html extension.
  4. If you opened the file in an editor's preview, try opening it in the browser directly: Right click → Open With → Chrome.
  5. If your server is serving the file with Content-Type: text/plain, serve it locally with: python3 -m http.server 8000 and open http://localhost:8000/otp-test.html.
  6. If you copied the HTML from a webpage that escaped tags (showing < and >), re-paste the raw HTML into a plain text file and save.
